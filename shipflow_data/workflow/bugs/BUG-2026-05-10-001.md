## Reproduction

- Preconditions:
  - The Flutter Android CI workflow runs on Blacksmith.
  - The Android overlay text injection helper is compiled as part of the debug APK.
- Steps:
  1. Push commit `06ddc57` or equivalent source to `master`.
  2. Run `.github/workflows/android-build.yml`.
  3. Wait for the `Build debug APK` step.

## Expected Behavior

- `flutter build apk --debug` should compile native Kotlin and produce `build/app/outputs/flutter-apk/app-debug.apk`.

## Observed Behavior

- `:app:compileDebugKotlin` fails because `AccessibilityNodeInfo.EXTRA_INPUT_TYPE` is not available.
- The workflow exits with code 1 before the debug APK artifact is created.

- `gh run view 25625957934 --repo diane-defores/winflowz_app --log-failed`
- CI excerpt: `OverlayTextInjectionHelper.kt:69:55 Unresolved reference 'EXTRA_INPUT_TYPE'`.
- Android documentation exposes `AccessibilityNodeInfo.getInputType()` from API 19, which matches the needed field classification.
