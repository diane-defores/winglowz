# Bug Record: BUG-2026-06-11-002

## Summary

- Title: Initial emoji recents panel is not horizontally scrollable
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-fix BUG-2026-06-11-002`

## Reproduction

- Preconditions:
  - Android IME opens with emoji panel on the initial `Recents` category.
- Steps:
  1. Open the WinFlowz keyboard in a text field.
  2. Tap the emoji mode.
  3. Try to drag horizontally on an emoji row before changing category.
  4. Change emoji category, then try to drag horizontally again.

## Expected Behavior

- The initial emoji panel should expose the horizontal scroll affordance when more emoji are available, and dragging the emoji rows should scroll immediately.

## Observed Behavior

- The initial emoji panel has no horizontal scroll affordance and does not scroll. After changing emoji category, the affordance appears and horizontal scroll works.

## Evidence

- Operator report in chat on 2026-06-11.
- Static diagnosis: initial `Recents` fallback was capped to exactly `expandedVisibleEmojiRows * visibleColumnsPerRow`, so each row had exactly one page and was not marked scrollable.
