# Bug Record: BUG-2026-05-24-001

## Summary

- Title: Validated onboarding cards still show the defer action
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-24-001 on Android APK`

## Reproduction

- Preconditions:
  - Android onboarding is visible.
  - At least one onboarding step is already satisfied, for example the WinFlowz keyboard is enabled.
- Steps:
  1. Open the onboarding overlay.
  2. View a card for an already satisfied setup item.
  3. Inspect the actions shown on that row.

## Expected Behavior

- A satisfied onboarding step does not show the `Plus tard` defer action.
- The satisfied row shows a non-actionable informational state labeled `Activ├®`.
- Configuration/edit affordances may remain available when the feature has settings to revisit.

## Observed Behavior

- Satisfied onboarding rows still rendered the generic defer action area, creating confusing copy on cards already validated by the user.
