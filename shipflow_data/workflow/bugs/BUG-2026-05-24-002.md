# Bug Record: BUG-2026-05-24-002

## Summary

- Title: Clipboard edit dialog crashes when closed
- Current status: `closed`
- Severity: `high`
- Owner: Codex
- Next command: none

## Reproduction

- Preconditions:
  - Android app session with clipboard screen available.
  - Manual clipboard item can be added.
- Steps:
  1. Add an item through the manual clipboard input field.
  2. Tap the edit icon on the created clipboard item.
  3. Press `Annuler`.
  4. Repeat the flow and press `Sauvegarder` without modifying content.

## Expected Behavior

- The edit dialog closes cleanly.
- `Annuler` leaves the item unchanged.
- `Sauvegarder` without a content change closes the dialog without crashing or freezing.

## Observed Behavior

- The app shows a red Flutter assertion screen and freezes/crashes.
- Reported assertion: `Failed assertion ... src/widgets/framework.dart, line 6268, position 12: dependents.isEmpty is not true.`

## Evidence

- Operator report in chat, 2026-05-24.
- No Sentry event ID was supplied.
