# Bug Record: BUG-2026-06-12-001

## Summary

- Title: Emoji panel closes despite Auto Close being disabled
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-06-12-001`

## Reproduction

- Preconditions:
  - Android IME is active.
  - Auto Close has been set to off.
- Steps:
  1. Open the WinFlowz keyboard in a text field.
  2. Open the emoji panel.
  3. Tap an emoji.

## Expected Behavior

- With Auto Close off, inserting an emoji should keep the emoji panel open.

## Observed Behavior

- The emoji panel closes after inserting an emoji.

## Evidence

- Operator report in chat on 2026-06-12.
- Static diagnosis: `autoCloseModeAfterTextInput()` already honors `autoCloseModesEnabled == false`, but `WinFlowzInputMethodService.onSharedPreferenceChanged()` did not refresh the active IME view when `KEY_AUTO_CLOSE_MODES_ENABLED` changed.
