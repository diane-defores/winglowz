# Bug Record: BUG-2026-05-19-003

## Summary

- Title: Clip action row activates without pin or explicit panel open
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sf-fix BUG-2026-05-19-003`

## Reproduction

- Preconditions:
  - Android IME enabled.
  - Clipboard allowed for the current field.
  - Clip action is not pinned.
- Steps:
  1. Open the WinFlowz keyboard in a text field.
  2. Open the symbols/signs mode.
  3. Press symbol keys.

## Expected Behavior

- Clip should show the full Clip page only after explicitly tapping `Clip`.
- Clip should show the compact action row only when explicitly pinned.
- Symbol key presses should not activate `Clip` or attach the Clip action row.

## Observed Behavior

- `Clip` can become active and display the Clip action row after unrelated symbol key presses.
- The action appears unpinned visually, but behaves like a pinned/attached action row.

## Evidence

- Operator report in chat on 2026-05-19.
- `WinFlowzInputMethodService.actionBarStateForCurrentField()` returns `stored.withAttachedClipboardActionRow()` for normal clipboard-allowed fields.
- `KeyboardActionCatalog.isActionActive("clipboard")` treats any attached clipboard row as active.
