Configurer Windows ne sert pas à "tweaker pour tweaker". Cela sert à réduire les micro-frictions qui grignotent ton attention toute la journée.

> Un bon setup Windows ne doit pas être impressionnant. Il doit être lisible, reproductible et agréable à utiliser.

## Le vrai sujet du module

Ce module n'a pas pour but de te transformer en collectionneur d'outils. Son but est plus simple :
- stabiliser la base
- enlever du bruit
- accélérer les gestes fréquents
- adapter Windows à ton vrai mode de travail

La bonne logique n'est donc pas :
- installer plein d'utilitaires

La bonne logique est :
- partir des frictions réelles
- choisir peu d'outils
- construire un environnement que tu pourras garder

## Le decision framework Winflowz

Quand tu veux améliorer ton poste Windows, pose-toi quatre questions :

1. **Quelle friction revient vraiment tous les jours ?**
2. **Est-ce un problème de réglage, d'outil, d'habitude, ou de matériel ?**
3. **Est-ce qu'une solution plus simple existe déjà dans Windows ?**
4. **Est-ce que ce changement rendra mon poste plus clair, ou seulement plus compliqué ?**

Ce cadre évite deux erreurs classiques :
- bricoler sans hiérarchie
- installer une nouvelle couche pour compenser un désordre de fond

## L'ordre juste pour améliorer Windows

Le bon ordre n'est pas de commencer par les outils les plus "fun". C'est de commencer par les couches qui soutiennent tout le reste.

### 1. Base système

Avant toute chose :
- stockage visible
- sauvegarde crédible
- installations propres
- machine suffisamment saine

Si cette couche est mauvaise, tout le reste repose sur du sable.

### 2. Navigateur

Le navigateur est souvent l'application la plus utilisée. Il mérite donc un vrai choix :
- navigateur principal
- stratégie d'onglets
- peu d'extensions, mais utiles

### 3. Ergonomie

Une fois la base propre, tu peux rendre le poste plus confortable :
- barre des tâches
- fenêtres
- petits ajustements de friction
- quelques outils ciblés si besoin réel

### 4. Automatisation

Tu n'automatises pas pour faire joli. Tu automatises quand un geste revient assez souvent pour justifier une règle, un raccourci ou un script.

### 5. Terminal

La ligne de commande n'est pas obligatoire. Elle devient utile quand :
- le travail est répétitif
- le volume augmente
- la reproductibilité compte

### 6. Tiling et gestion de fenêtres

Le bon niveau de window management dépend de ton profil :
- snap natif
- FancyZones
- outils intermédiaires
- vrais gestionnaires avancés seulement si tu as le besoin

## Ce que couvre ce module

- **Santé système** pour rendre la base lisible et fiable
- **Navigateurs** pour choisir un vrai poste web
- **Ergonomie** pour enlever les frictions visibles
- **Automatisation** pour réduire les gestes répétitifs
- **Terminal** pour les tâches textuelles, massives ou scriptables
- **Tiling** pour adapter l'organisation des fenêtres à ton style de travail

## L'état d'esprit utile

Le bon réflexe n'est pas :
- "quel outil cool puis-je ajouter ?"

Le bon réflexe est :
- "quelle friction est-ce que je supprime ?"

Un bon poste Windows productif ressemble à ça :
- peu de bruit
- peu de couches inutiles
- quelques bons raccourcis
- des outils choisis pour un rôle précis
- un système que tu peux comprendre et reconstruire

## Workflow recommandé

**Minimaliste** :
- base système propre
- un bon navigateur
- ergonomie sobre

**Pragmatique** :
- quelques couches ciblées
- automatisation des gestes récurrents
- terminal pour les tâches qui le méritent

**Système personnel** :
- poste documenté
- choix reproductibles
- gestion des fenêtres et des outils adaptée à ton profil réel

:::note[Exercice pratique]
Avant de lire le reste du module, note :

1. les 3 frictions Windows qui reviennent le plus souvent
2. lesquelles relèvent du système, du navigateur, de l'ergonomie ou de l'automatisation
3. laquelle mérite d'être traitée en premier

Si tu n'arrives pas à nommer tes frictions, tu risques surtout d'empiler des outils sans améliorer ton poste.
:::
