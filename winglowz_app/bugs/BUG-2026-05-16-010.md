# Bug Record: BUG-2026-05-16-010

## Summary

- Title: Media Now/App buttons do not guide users to required media access permission
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-16-010`

## Reproduction

- Preconditions:
  - Android real device.
  - WinGlowz keyboard active.
  - A media app is playing sound.
  - WinGlowz media session/notification listener access is not enabled.
- Steps:
  1. Open the Media action bar.
  2. Tap `Now`.
  3. Tap `App`.

## Expected Behavior

- If media session access is missing, `Now`/`App` should route the user to WinGlowz onboarding with a clear media access step and a button that opens the Android notification listener settings.
- Once enabled, `Now` should show current media metadata and `App` should open the app currently playing audio.

## Observed Behavior

- `Now` only displays a dead-end message asking to enable media access.
- `App` does not open the current media app when media session access is missing.
- Other transport controls can still work because media key dispatch does not require session metadata access.

- User confirmation in chat at 2026-05-16 13:53:53 UTC.
