# BUG-2026-05-16-012 - Android QA regressions after action/media/compact work

Status: fixed-pending-verify
Date: 2026-05-16
Area: Android keyboard / diagnostics / compact panels / app-specific editing

## Evidence
Manual Android QA reported:
- `DelW→` works in Termux, SMS, browser, email, but in Obsidian Android deletes one letter left instead of one word right.
- `Début` and `Fin` fail in Termux.
- Compact mode fails for navigation, media, clipboard, signs, accents panels.
- Diagnostic text does not include media session/write-settings permission fields.
- Termux Copy/Cut still fail.

## Fixes
- Diagnostic output now includes `media_session_access` and `system_settings_write`.
- Compact panel mode now suppresses typing rows when any panel is open.
- Navigation panel gets a compact 2-row horizontally scrollable layout.
- Obsidian `DelW→` now avoids the direct `deleteSurroundingText` path and sends Ctrl+ForwardDelete instead.
- Termux `Début`/`Fin` now send Ctrl+A/Ctrl+E.

## Known limitation
Termux Copy/Cut remain unresolved. Paste works. Copy/Cut through IME shortcuts are target-app dependent and Termux does not expose the same copy/cut behavior as the touch selection menu.

## Manual QA
Partial retest: Obsidian delete left/right PASS; Termux Début/Fin PASS; Termux DelW→ PASS; Termux DelW← failed and was patched to Ctrl+W. Next APK verification pending.
