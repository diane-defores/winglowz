# Bug Record: BUG-2026-05-16-006

## Summary

- Title: Native keyboard main action bar became scrollable and contextual long press stopped attaching rows
- Current status: `fixed-pending-verify`
- Severity: `high`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-16-006`

## Reproduction

- Preconditions:
  - Android real device.
  - WinGlowz keyboard active.
  - Build `sha=0cd6d85`, run `25961717167` or equivalent action-bar architecture build.
- Steps:
  1. Open any text field with the WinGlowz keyboard.
  2. Observe the main action bar above the letters.
  3. Long press `123`, `Prefs`, and `Nav`.
  4. Try horizontal scrolling on the main action bar and on contextual rows.

## Expected Behavior

- Main action bar keeps the previous compact behavior: all primary action buttons distributed across the available width, not replaced by oversized horizontally scrolling buttons.
- If horizontal paging is later added to the main action bar, it must be page-by-page, not free pixel scrolling.
- Long press on action buttons with contextual rows attaches/displays the row (`123`, `Nav`, `Clip`, `Media`).
- Context rows can page horizontally as whole pages.

## Observed Behavior

- Main action bar became horizontally scrollable with wider buttons, reducing immediate access to actions.
- Long press `123` appears to do nothing.
- Long press `Prefs` shows `Pin unavailable`.
- Long press `Nav` shows `Action pinned`, but no contextual row appears.
- Keyboard effects/shadows appear clipped near the action bar boundaries.
