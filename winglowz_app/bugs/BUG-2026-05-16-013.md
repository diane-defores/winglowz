# Bug Record: BUG-2026-05-16-013

## Summary

- Title: Android keyboard action rows cannot be consistently pinned and unpinned
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: retest on Android device

## Reproduction

- Preconditions:
  - WinGlowz Android keyboard active.
  - Action bar visible.
- Steps:
  1. Long-press `123`, `Nav`, or `Media` to attach/pin an action row.
  2. Long-press the same action again.
  3. Try to long-press `#+=`, `Emoji`, `Acc`, `Clip`, or `Snip`.

## Expected Behavior

- Long press on a pinnable action with a row toggles it:
  - first long press pins and shows the row;
  - second long press unpins and removes the row.
- `#+=`, `Emoji`, `Acc`, `Clip`, `Snip`, and `Media` expose useful pinnable action rows when allowed by field policy.

## Observed Behavior

- Existing attached rows could not be removed by a second long press.
- Some actions had no row provider, so long press could not pin a useful row.
- The old minimal pinned recovery included actions that should be user-controlled, making the visual pin state confusing.

## Evidence

- User report on 2026-05-16.
- Code inspection of `KeyboardActionBarController.handleAttachRowLongPress`, `KeyboardActionCatalog`, and `KeyboardStateStore.DEFAULT_ACTION_BAR_LONG_PRESS_BEHAVIOR`.
