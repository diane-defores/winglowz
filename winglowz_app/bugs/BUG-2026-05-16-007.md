# Bug Record: BUG-2026-05-16-007

## Summary

- Title: Android keyboard Delete word left does not delete the previous word
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-16-007`

## Reproduction

- Preconditions:
  - Android real device.
  - WinGlowz keyboard active.
  - Text field contains text before the cursor.
- Steps:
  1. Open the Navigation panel or attached navigation row.
  2. Tap `DelW←` / Delete word left.

## Expected Behavior

- The word immediately before the cursor is deleted.

## Observed Behavior

- `Début` works.
- `Fin` works.
- Delete word right works.
- Delete word left does not delete the previous word.

## Evidence

- User confirmation in chat at 2026-05-16 13:28:37 UTC.

## Diagnosis Notes

| Date UTC | Skill | Author | Hypothesis | Findings | Outcome |
|----------|-------|--------|------------|----------|---------|
| 2026-05-16 13:28:37 UTC | sf-fix | GPT-5 Codex | `DeleteWordBefore` sends `Ctrl+Backspace` first, and some Android fields accept the key event without actually deleting text, preventing the fallback path. | `onDeleteWordBefore` returned immediately when `sendSoftKey(KEYCODE_DEL, CTRL)` reported success, before using text-before-cursor deletion. | supported |
