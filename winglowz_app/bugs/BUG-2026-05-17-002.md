# Bug Record: BUG-2026-05-17-002

## Summary

- Title: Settings keyboard preview does not show action bar or real pin badge and keeps bottom empty row
- Current status: `open`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sf-fix BUG-2026-05-17-002`

## Reproduction

- Preconditions:
  - WinGlowz Android app available with current local changes.
  - User can open Settings keyboard preview and Keyboard Theme Studio.
- Steps:
  1. Open the keyboard preview from Settings.
  2. Inspect the top of the keyboard preview.
  3. Inspect the bottom of the keyboard preview.
  4. Open Keyboard Theme Studio and switch between themes.
  5. Inspect the pin badge rendered in the Theme Studio preview.

## Expected Behavior

- Settings keyboard preview shows the action bar before the QWERTY typing rows.
- Settings keyboard preview does not leave an empty row/gap at the bottom.
- Previewed pinned actions show a badge that is representative enough to validate the current theme, not only a generic dot.
- Theme Studio preview renders a theme-aware pin badge close to the real native keyboard badge behavior.

## Observed Behavior

- User reports the settings keyboard preview still has an empty bar/row at the bottom.
- User reports the settings keyboard preview starts directly at QWERTY.
- User reports the action bar is not visible in the preview.
- User confirms real Android keyboard theme pin badges are functional across themes.
- User reports Theme Studio preview shows only a generic dot on `D`, not the real/preset-specific pin badge.
