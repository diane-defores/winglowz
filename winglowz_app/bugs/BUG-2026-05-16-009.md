# Bug Record: BUG-2026-05-16-009

## Summary

- Title: Copy and Cut actions do not work inside Termux
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-16-009`

## Reproduction

- Preconditions:
  - Android real device.
  - WinGlowz keyboard active in Termux.
  - Termux has text selected or an app/editor state where copy/cut should be meaningful.
- Steps:
  1. Tap `Clip` or open clipboard actions.
  2. Tap `Copy`.
  3. Tap `Cut`.

## Expected Behavior

- Copy should place selected Termux text into Android clipboard when Termux exposes or handles the selection.
- Cut should remove/copy selected text when Termux or the active terminal app supports it.
- Existing behavior outside Termux should remain unchanged.

## Observed Behavior

- Paste works in Termux.
- Copy and Cut work outside Termux.
- Copy and Cut do not work inside Termux.

## Evidence

- User confirmation in chat at 2026-05-16 13:36:55 UTC.
- Local audit of `/home/claude/keyboard`: Unexpected Keyboard uses Android context menu actions for copy/cut/paste; user reports it also does not solve Termux copy/cut.
