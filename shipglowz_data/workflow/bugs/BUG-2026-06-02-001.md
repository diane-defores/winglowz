# Bug Record: BUG-2026-06-02-001

## Summary

- Title: Closed Android overlay bubble cannot be dragged
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-fix BUG-2026-06-02-001`

## Reproduction

- Preconditions:
  - Android overlay permission is granted.
  - WinGlows floating overlay is visible in the collapsed sphere state.
- Steps:
  1. Open or show the WinGlows overlay.
  2. Return it to the collapsed sphere state.
  3. Try to drag the sphere to another screen position.

## Expected Behavior

- The collapsed sphere can be dragged to a new position without opening the recording controls.
- A short tap on the sphere still starts dictation.
- The new position is clamped to the visible screen and saved like expanded overlay drag.

## Observed Behavior

- Dragging works from the expanded overlay handle.
- In collapsed state, the small sphere does not move as expected.

## Evidence

- `OverlayForegroundService.attachTouchListener` attached drag handling to the expanded drag handle and the parent overlay view.
- `OverlayView` kept the collapsed `fabView` as a child with its own click listener, so touches on the sphere did not reliably use the parent drag handler.
