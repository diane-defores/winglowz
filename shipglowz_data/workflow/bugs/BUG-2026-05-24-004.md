# Bug Record: BUG-2026-05-24-004

## Summary

- Title: Action row Mini height is far below one third
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: Run Android IME QA through Blacksmith APK and Diane physical-device validation.

## Reproduction

- Preconditions:
  - Android IME keyboard preferences are available.
  - Keyboard is in normal mode.
- Steps:
  1. Open keyboard preferences.
  2. Set Action row height to `Mini`.
  3. Open/use action rows, including horizontally scrollable action rows.

## Expected Behavior

- `Full` uses the full action-row height.
- `Square` renders at about two thirds of `Full`.
- `Mini` renders at about one third of `Full` and remains touchable.

## Observed Behavior

- `Mini` appears closer to one fifth of a normal key height in normal mode.
- The rendered buttons are too small to use comfortably.

## Evidence

- Operator report in chat on 2026-05-24.
- Static diagnosis: scrollable action rows additionally shrink visual height by 30%, so the old Mini scale `0.30` can render around `0.21` of full height.
