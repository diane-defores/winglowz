# Bug Record: BUG-2026-05-19-001

## Summary

- Title: Keyboard mic uses Android fallback without enough visible fallback explanation
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: codex
- Next command: `/sf-ship BUG-2026-05-19-001`

## Reproduction

- Preconditions:
  - Android real device.
  - Installed debug APK reports `build: sha=37116dd | run=26091472372 | ref=master`.
  - WinGlows keyboard is enabled and active.
  - No local ASR pack is installed for active language `fr-FR`.
  - `allow_cloud_fallback=false`.
- Steps:
  1. Open any text field and show the WinGlows keyboard.
  2. Press the `Mic` button in the keyboard action bar.
  3. Observe what mode/fallback information is shown before or during dictation.
  4. Copy Settings diagnostics after the attempt.

## Expected Behavior

- The keyboard/action-bar mic flow should make the runtime mode understandable before capture starts or immediately when fallback starts.
- With no local pack installed, WinGlows should propose a local pack install or explicitly label Android fallback/unavailable.
- With `allow_cloud_fallback=false`, there must be no cloud fallback and the user should be able to tell that cloud is off.

## Observed Behavior

- Pressing `Mic` in the keyboard action bar launched Android dictation and dictation worked.
- The user did not see cloud/fallback/mode information in the flow, although the user inferred it was Android fallback.
- Diagnostic after the test shows no cloud fallback was active:
  - `allow_cloud_fallback=false`
  - `voice_runtime=unavailable`
  - `voice_language=fr-FR`
  - `voice_pack=none`
  - `voice_engine=android_speech_recognizer`
  - `voice_fallback_reason=missing_pack`
  - `voice_last_error_code=none`
