# Bug Record: BUG-2026-06-12-002

## Summary

- Title: Android overlay start no longer shows the floating widget
- Current status: `fix-attempted`
- Severity: `high`
- Owner: Diane
- Next command: `/107-sf-test`

## Reproduction

- Preconditions:
  - Android device with WinGlowz overlay feature available.
  - Accessibility permission may already be enabled.
- Steps:
  1. Open WinGlowz and go to a screen that can start the overlay.
  2. Tap the overlay start action.
  3. Observe whether the floating overlay widget appears.

## Expected Behavior

- Starting overlay should either:
  - request microphone permission when it is missing, then show the floating widget after grant, or
  - show a clear permission denial result without leaving the overlay session locked.

## Observed Behavior

- The floating widget no longer appears on start.
- Recent native changes introduced a path where overlay launch is blocked before the widget can be shown.

## Evidence

- Operator report in chat on 2026-06-12.
- `MainActivity.startOverlayRecording` rejected launch immediately on missing `RECORD_AUDIO` instead of requesting the runtime permission.
- `OverlayForegroundService.handleStart` did not release `MicrophoneSessionCoordinator` when startup failed before `running=true`.
