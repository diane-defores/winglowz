# Bug Record: BUG-2026-05-20-001

## Summary

- Title: Keyboard mic Android fallback stops before user presses mic again
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: codex
- Next command: `/sf-ship BUG-2026-05-20-001`

## Reproduction

- Preconditions:
  - Android real device.
  - WinGlows keyboard is enabled and active.
  - No local ASR pack is installed for the active language.
  - Keyboard action-bar Mic uses Android SpeechRecognizer fallback.
- Steps:
  1. Open a text field and show the WinGlows keyboard.
  2. Press the action-bar `Mic` button.
  3. Observe the recording animation and status bar.
  4. Keep dictating or pause briefly without pressing Mic again.

## Expected Behavior

- The first Mic tap starts an active dictation session.
- The session remains active until the user presses Mic again, Stop, Pause, Cancel, or the input field/session ends.
- Android SpeechRecognizer fallback may segment internally, but WinGlows should keep the user-facing session active and only insert the accumulated text on explicit user stop.

## Observed Behavior

- Recording animation starts.
- Status bar shows `Android speech fallback: listening (missing_pack)`.
- Recording stops after a few seconds without the user pressing Mic again.

## Evidence

- User report in chat on 2026-05-20.
- No secrets, audio content, transcript content, tokens, or screenshots were provided or persisted.
