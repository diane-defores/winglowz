# Bug Record: BUG-2026-05-12-001

## Summary

- Title: Firestore rules deploy cannot impersonate Google service account
- Current status: `closed`
- Severity: `high`
- Owner: codex
- Next command: none

## Reproduction

- Preconditions:
  - GitHub repository has `GCP_WIF_PROVIDER`, `GCP_WIF_SERVICE_ACCOUNT`, and `FIREBASE_PROJECT_ID` secrets configured.
  - Workflow runs the `firebase` job in `.github/workflows/android-build.yml`.
- Steps:
  1. Trigger `Flutter Android CI` on `main`, `master`, or `workflow_dispatch`.
  2. Let the `Deploy Firestore Rules and Indexes` job reach `Authenticate to Google Cloud via OIDC`.

## Expected Behavior

- GitHub Actions obtains short-lived Google credentials through Workload Identity Federation.
- Firebase CLI deploys Firestore rules and indexes to the configured Firebase project.

## Observed Behavior

- `google-github-actions/auth` fails before the Firebase deploy step.
- Error states `Permission 'iam.serviceAccounts.getAccessToken' denied on resource (or it may not exist)`.

## Evidence

- User-provided annotation, redacted by GitHub secret masking.
- `.github/workflows/android-build.yml` previously requested a generated access token from `google-github-actions/auth@v3` and passed it to Firebase as `FIREBASE_TOKEN`.
