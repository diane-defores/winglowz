# Bug Record: BUG-2026-06-11-001

## Summary

- Title: Keyboard vibration mode is not preserved and long mode feels too short
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-06-11-001`

## Reproduction

- Preconditions:
  - WinGlowz Android IME is enabled.
  - The keyboard settings panel is reachable.
- Steps:
  1. Set keyboard vibration to `long`.
  2. Type on the native keyboard and compare the felt duration with the mode name.
  3. Change another keyboard preference from Flutter Settings.
  4. Return to the native keyboard vibration mode.

## Expected Behavior

- Each vibration mode should map to a clearly distinct haptic duration.
- `long` should feel materially longer than `short`.
- Saving unrelated keyboard preferences from Flutter should preserve the native vibration level.

## Observed Behavior

- `long` still feels short in practice.
- Flutter-driven preference saves can collapse the native vibration level because only the boolean enabled flag is sent back.

## Evidence

- User report in chat on 2026-06-11.
- `MainActivity.setKeyboardPreferences` consumed `keyVibrationEnabled` but not `keyVibrationIntensity`.
- `AndroidKeyboardBridge.setPreferences` and `AndroidKeyboardStatus` exposed only the boolean vibration state.
- `WinGlowzKeyboardView.performKeyboardHaptic` handled `short` and `medium` identically and used `LONG_PRESS` for `long`.
