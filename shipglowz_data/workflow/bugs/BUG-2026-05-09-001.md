# Bug Record: BUG-2026-05-09-001

## Summary

- Title: CI warning filter reports expected defaults as warnings
- Current status: `fix-attempted`
- Severity: `low`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-09-001`

## Reproduction

- Preconditions:
  - The Flutter Android CI workflow runs on a fresh Blacksmith Ubuntu runner.
  - The log view is filtered to warning/error messages.
- Steps:
  1. Run the `Analyze, Test, Build APK` and `Supabase Migration Tests` jobs.
  2. Open logs with `level:warn,error`.
  3. Review messages around checkout, artifact upload, and Supabase startup.

## Expected Behavior

- CI logs should not report expected defaults or intentionally empty project state as warnings.
- Missing debug APK artifacts should still fail the build before upload.
- Supabase migration tests should start without warning about a missing default seed file.

## Observed Behavior

- `actions/upload-artifact` exposed the default `if-no-files-found: warn` behavior in logs.
- Supabase CLI warned that no files matched `supabase/seed.sql`.
- Git printed default-branch hints during fresh repository initialization.

## Evidence

- Chat-provided CI log excerpt from May 8, 2026 20:45-21:33 UTC.
- `.github/workflows/android-build.yml` used checkout as the first step in both jobs.
- `.github/workflows/android-build.yml` did not set `if-no-files-found`.
- No `supabase/seed.sql` file existed before this fix attempt.
- Supabase seeding documentation says the CLI looks for `supabase/seed.sql` by default when no specific seed configuration is provided.
- `actions/upload-artifact` documentation lists `warn`, `error`, and `ignore`, with `warn` as the default.
