# Bug Record: BUG-2026-05-17-001

## Summary

- Title: Corner editor reset buttons leave preset corners visible
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sf-fix BUG-2026-05-17-001`

## Reproduction

- Preconditions:
  - Keyboard corner editor has an active preset with visible corner shortcuts.
- Steps:
  1. Open the keyboard corner shortcut editor.
  2. Select a key that has visible preset corners.
  3. Press `Reset corner` or `Reset key`.

## Expected Behavior

- The selected corner, or all corners on the selected key, disappear from the keyboard preview immediately in the draft.

## Observed Behavior

- The buttons appear to do nothing visually because preset-provided corners are resolved again after overrides are removed.

## Evidence

- Static diagnosis in `lib/features/keyboard/domain/keyboard_models.dart`: `resetCorner` and `resetKey` remove overrides only.
- Static diagnosis in `KeyboardCornerPresetCatalog.resolvedForKey`: preset shortcuts are resolved before overrides, so removing an override reveals the preset shortcut.
