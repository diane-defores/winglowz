# Bug Record: BUG-2026-05-10-002

## Summary

- Title: Installed Android app shows blank main tab content
- Current status: `closed`
- Severity: `high`
- Owner: codex
- Next command: `closed`

## Reproduction

- Preconditions:
  - User opens the authenticated/local app shell.
- Steps:
  1. Open the app.
  2. Navigate through Voice, Clipboard, Snippets, and Dictionary.
  3. Compare the central page area with Settings.

## Expected Behavior

- Each tab should show its main content in the central area without requiring the onboarding block to be cleared first.

## Observed Behavior

- The onboarding panel appears at the top of every page, the bottom navigation remains visible, and the central area appears empty on the functional tabs except Settings.
- After the overlay/onboarding change, the installed APK still shows no central content on the functional tabs while Settings renders and scrolls normally.
- User copied Settings diagnostic: `legacy_supabase: false`, `status: local_mode`, missing `SUPABASE_URL` and `SUPABASE_PUBLISHABLE_KEY`.
- Later diagnostics captured the concrete Flutter render failure: functional tabs hit infinite-width button constraints, which prevented tab bodies from laying out.

## Evidence

- User report in chat on 2026-05-10.
- `lib/features/shell/presentation/app_shell_screen.dart` renders `_OnboardingPanel` above every selected page.
- User-provided Settings diagnostic v4 on 2026-05-11 with `BoxConstraints(w=Infinity, 44.0<=h<=Infinity)`, repeated `RenderBox was not laid out`, and `screen_build` events for Clipboard, Snippets, Dictionary, and Settings.
- Blacksmith Android CI run `25662735886` on 2026-05-11 passed and uploaded `winglowz_app-debug-apk`.
