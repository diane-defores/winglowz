# Security and Reliability with WinFlowz

In an era where data breaches make headlines daily, building trust through robust security isn't optional — it's foundational.

## Local-First Architecture

One of WinFlowz's core design principles is **local data processing**. Our Obsidian plugins (RSSFlowz, PluginFlowz) process everything on your device. Your notes, your feeds, your configurations — none of it leaves your machine.

This means:
- No cloud dependency for core functionality
- No third-party access to your content
- Full control over your data at all times

## API Key Management

Some of our tools integrate with third-party services (YouTube for TubeFlowz, RSS feeds for RSSFlowz). In these cases:
- **You** provide and manage your own API keys
- Keys are stored locally in your Obsidian vault settings
- WinFlowz never transmits or stores your API credentials

## Reliable by Design

Our tools are built for professionals who can't afford downtime:
- **Offline capability:** Core features work without internet
- **Graceful degradation:** If an API is unavailable, the tool continues working with cached data
- **Version pinning:** Plugin updates are opt-in, so a new version never breaks your workflow unexpectedly

## Best Practices We Recommend

1. **Keep plugins updated** — security patches are included in every release
2. **Use separate API keys** per tool when possible
3. **Back up your vault** regularly (we recommend Git or a local backup solution)
4. **Review permissions** before installing any Obsidian community plugin

Security isn't a feature — it's a commitment. We take it seriously so you can focus on being productive.
