# Sources — WinGlowz

## Recherche sur la productivité

- **Cal Newport** — *Deep Work* : méthodologie de travail concentré, fondement théorique de l'approche WinGlowz
- **James Clear** — *Atomic Habits* : formation d'habitudes productives, applicable à la gamification
- **David Allen** — *Getting Things Done (GTD)* : système d'organisation référence pour les professionnels

## Données de marché

- **StatCounter** — Parts de marché des systèmes d'exploitation desktop (validation du positionnement Windows 75%+)
- **Gartner** — Rapports sur la productivité en entreprise, tendances du travail hybride
- **Statista** — Statistiques sur l'adoption des outils de productivité

## Écosystème Windows

- **Microsoft Dev Blog** — Annonces et évolutions de la plateforme Windows
- **Windows Insider Program** — Fonctionnalités à venir, anticipation des tendances
- **PowerToys GitHub** — Repository open source, roadmap, retours communautaires

## Recherche UX

- **Nielsen Norman Group** — Études d'utilisabilité, heuristiques de design, bonnes pratiques UX
- **Baymard Institute** — Recherche sur l'expérience utilisateur, benchmarks d'interface

## Documentation technique

- **Astro** — [docs.astro.build](https://docs.astro.build) : framework SSG, View Transitions, intégrations
- **Vue 3** — [vuejs.org](https://vuejs.org) : Composition API, composants SFC, réactivité
- **Starlight** — [starlight.astro.build](https://starlight.astro.build) : thème documentation Astro
- **UnoCSS** — [unocss.dev](https://unocss.dev) : moteur CSS utilitaire
- **Clerk** — [clerk.com/docs](https://clerk.com/docs) : authentification et gestion utilisateurs
- **Convex** — [docs.convex.dev](https://docs.convex.dev) : backend temps réel

## Communautés

- **r/Windows** — Subreddit principal de la communauté Windows
- **r/productivity** — Discussions et recommandations d'outils de productivité
- **Hacker News** — Veille technologique, lancement de produits
- **Product Hunt** — Découverte de nouveaux outils, benchmark de lancements
