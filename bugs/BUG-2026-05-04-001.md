## Summary

- Title: Android build asks for legacy Supabase anon key
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-04-001`

## Reproduction

- Preconditions:
  - Android app installed without current Supabase runtime defines.
- Steps:
  1. Launch installed VoiceFlowz Android app.
  2. App reaches auth gate before Supabase initialization.
  3. Read the displayed missing configuration message.

## Expected Behavior

- The app asks for `SUPABASE_URL` and `SUPABASE_PUBLISHABLE_KEY`.
- The docs and examples use the current Supabase publishable key naming.
- Legacy `SUPABASE_ANON_KEY` remains an internal compatibility fallback only.

## Observed Behavior

- The app asks for `SUPABASE_URL` or `SUPABASE_ANON_KEY`.
- README, `.env.example`, CLAUDE and task/docs wording still use the old env var name.

## Evidence

- `lib/core/bootstrap/supabase_bootstrap.dart` read `SUPABASE_ANON_KEY`.
- `README.md`, `.env.example`, `CLAUDE.md`, `docs/ARCHITECTURE_FLUTTER.md`, `TASKS.md` referenced the old name.
- Context7 Supabase docs query on 2026-05-04 returned current Flutter quickstart examples using "publishable key"; local `supabase_flutter` 2.12.4 still names its SDK parameter `anonKey`.
