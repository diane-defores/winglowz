# Bug Record: BUG-2026-05-16-003

## Summary

- Title: Compact keyboard bottom row is overlapped by Android navigation bar
- Current status: `fix-attempted`
- Severity: `high`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-16-003`

## Reproduction

- Preconditions:
  - Android real device with system bottom/navigation bar visible.
  - WinFlowz keyboard active.
  - Compact mode enabled through height reduction or prefs.
- Steps:
  1. Open any text field with WinFlowz keyboard.
  2. Reduce keyboard height until compact mode is enabled, or enable compact mode from prefs.
  3. Try using the bottom row keys.

## Expected Behavior

- Compact mode remains fully usable above the Android bottom/navigation bar.
- Bottom-row keys must not be hidden or overlapped.
- User requested prefs behavior: the large `COMPACT` button should be a toggle; it should keep showing the current height when using `H+/-`, and pressing it should switch compact mode.

## Observed Behavior

- Compact keyboard behavior works, but the bottom of the keyboard is overlapped by the Android bottom bar.
- The overlapped area makes compact mode unusable on the real device.

## Evidence

- `shipflow_data/workflow/TEST_LOG.md` entry `2026-05-16 - Keyboard crash recovery Android real-device QA`.
- User report: "en compact tout fonctionne actuellement mais le bas du clavier et overlapped par la bottom bar android ! donc inutilisable."
