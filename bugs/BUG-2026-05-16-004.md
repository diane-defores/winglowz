# Bug Record: BUG-2026-05-16-004

## Summary

- Title: Settings Backend Provider logs panel crashes with Flutter assertion
- Current status: `closed`
- Severity: `high`
- Owner: Diane
- Next command: `closed`

## Reproduction

- Preconditions:
  - Open WinFlowz Settings.
  - Backend Provider section is collapsed.
- Steps:
  1. Tap the chevron/header for `Backend Provider`.
  2. Inspect the logs/diagnostic panel.
  3. Scroll the settings page.

## Expected Behavior

- Backend Provider expands normally.
- Diagnostic logs render in a bounded readable panel.
- The page remains scrollable to the bottom.

## Observed Behavior

- A red Flutter error panel appears with yellow assertion text: `Failed assertion ... children.contains(element) is not true` / framework assertion.
- The red panel appears unbounded, preventing normal scrolling to the bottom.

## Evidence

- User report in chat on 2026-05-16.
- Widget regression test added: `settings backend diagnostics panel opens without layout error`.
