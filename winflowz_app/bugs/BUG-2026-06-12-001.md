# Bug Record: BUG-2026-06-12-001

## Summary

- Title: IME sound preference cycles invisibly and preview sounds feel duplicated
- Current status: `fix-attempted`
- Severity: `medium`
- Owner: unassigned
- Next command: retest on Android device

## Reproduction

- Preconditions:
  - WinFlowz Android keyboard active.
  - Preferences/settings panel visible inside the IME.
- Steps:
  1. Tap the `Sound` button repeatedly.
  2. Observe the button state and listen to the preview sound.

## Expected Behavior

- Each tap advances clearly to the next sound mode.
- The active mode is visible on the button.
- The preview plays once and sounds materially different across modes.

## Observed Behavior

- The button remained visually stuck on `Sound ON` across several taps.
- The different sound modes were not named, so the cycle looked broken.
- Some previews sounded like two rapid plays or a cut-off/doubled sound.

## Evidence

- User report on 2026-06-12.
- Code inspection of the IME settings key and native sound preview flow.

## Diagnosis Notes

| Date UTC | Skill | Author | Hypothesis | Findings | Outcome |
|----------|-------|--------|------------|----------|---------|
| 2026-06-12 00:00:00 UTC | sf-fix | GPT-5 Codex | The UI looks stuck because the `Sound` key only exposes ON/OFF while the underlying native state actually cycles through multiple intensities, and the preview sounds doubled because some profiles are multi-tone sequences. | `ToggleKeySound` cycled `OFF -> SHORT -> MEDIUM -> LONG -> EXTRA -> OFF`, but the key label only showed `Sound on/off`. The native profiles for medium/long/extra were encoded as 2-3 sequential tones, which plausibly matches the reported doubled/cut sound. | supported |
